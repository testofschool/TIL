### CLAUDE.md vs Hook — 실행 구조의 근본적 차이

> CLAUDE.md에 "Prettier 돌려"라고 적으면 80% 실행. Hook으로 PostToolUse에 걸면 100% 실행.

| 구분 | CLAUDE.md | Hook |
|------|-----------|------|
| 본질 | context window 안의 텍스트 (제안) | 모델 판단 무관하게 실행되는 스크립트 (법) |
| 실행률 | ~80% | 100% |
| 시점 | 모델이 읽고 판단 | PreToolUse/PostToolUse/Stop |
| 제어 | exit code 없음 | exit 2 = 차단, exit 0 = 통과 |
| 공유 | git 체크인 | `.claude/settings.json` git 체크인 |

**실전 Hook 8개 조합:**

PreToolUse 4개 (경비원):
1. Block Dangerous Commands: `rm -rf`, `git reset --hard`, `DROP TABLE` 패턴 매칭 → exit 2 차단
2. Protect Sensitive Files: `.env`, `package-lock.json`, `*.pem` 수정 원천 차단
3. Require Tests Before PR: 테스트 미통과 시 PR 생성 차단 (matcher: `mcp__github__create_pull_request`)
4. Log Every Command: 모든 bash 명령을 타임스탬프와 함께 `.claude/command-log.txt`에 기록

PostToolUse 3개 (검수 라인):
5. Auto-Format: Prettier(JS) / Black(Python) / gofmt(Go) 자동 실행
6. Auto-Lint: ESLint 실행 → 에러 시 에이전트가 직접 수정
7. Auto-Test: 테스트 실행 → 실패 시 즉시 수정, `tail -5`로 context window 오염 방지

Stop 1개 (결과 보존):
8. Auto-Commit: 에이전트 응답 완료 시 `git add -A && git commit`

**체이닝 순서가 핵심:** Prettier → ESLint → 테스트. 사람이 코드를 보는 시점에 이미 포맷+린트 통과 상태. 각 Hook이 200ms 안에 끝나는지가 관건.
