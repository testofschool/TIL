## 심화과정 사전평가 개념 정리

### 1. 동기(Synchronous) vs 비동기(Asynchronous)
- 동기: 한 작업이 완료될 때까지 다음 작업이 블로킹됨
- 비동기: 작업 완료를 기다리지 않고 다른 작업 수행 가능. 완료 시 callback이나 await로 처리
- 비유: 동기 = 카운터 앞에서 커피 기다리기, 비동기 = 진동벨 받고 다른 일 하기

### 2. HTTP 메서드 (CRUD)
- POST = Create (생성) — 회원가입, 글 작성
- GET = Read (조회) — 목록 불러오기, 데이터 조회
- PUT = Update (수정) — 프로필 수정, 글 수정
- DELETE = Delete (삭제) — 게시글 삭제

### 3. RESTful API
- API = 프로그램 간 대화 창구
- REST = 이 창구를 설계하는 규칙
- URL은 명사(자원): `/creators`, `/posts/3`
- HTTP 메서드가 동사(행위): GET, POST, PUT, DELETE
- URL에 동사 넣으면 안 됨 (❌ `/getCreators` → ✅ `GET /creators`)

### 4. WAS vs WS
- WS (Web Server): 정적 파일(HTML, CSS, 이미지) 전달. 예: Nginx, Apache
- WAS (Web Application Server): 로직 실행 후 동적 응답 생성. 예: Tomcat, FastAPI, Node.js
- 실무 구조: WS(앞단) → 정적 요청은 직접 처리, 동적 요청은 WAS로 전달

### 5. 학습 / 검증 / 테스트 세트
- 나누는 이유: 과적합(Overfitting) 방지
- 과적합 = 학습 데이터를 외워버려서 새 데이터에서 성능 하락
- 학습(Training): 모델이 공부하는 데이터
- 검증(Validation): 학습 중 중간 점검 + 얼리 스타핑 판단 기준
- 테스트(Test): 한 번도 안 본 데이터로 최종 성능 평가 (딱 1번)

### 6. CNN (합성곱 신경망)
- 이미지에서 특징을 추출하는 신경망
- 합성곱(Convolution): 필터가 이미지 위를 슬라이딩하며 패턴 추출
- 풀링(Pooling): Max Pooling으로 핵심 정보 유지하며 압축
- 완전연결(Fully Connected): 추출된 특징 종합하여 최종 분류

### 7. RNN (순환 신경망)
- 순서가 있는 시퀀스 데이터를 처리하는 신경망
- 이전 단계의 출력을 다음 단계 입력으로 순환시켜 문맥 기억
- 자연어 처리 진화: BoW → TF-IDF → Embedding → **RNN** → Transformer
- RNN의 한계: 긴 시퀀스에서 장기 의존성 문제 (앞부분을 까먹음)
- → Transformer(GPT, BERT)가 이 문제를 해결
