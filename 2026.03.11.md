오늘의 TIL (Today I Learned) — 2026.03.11
1. 아이디어 공장의 근본적 결함을 발견했다
4개 공식 문서에 퀴즈 구현이 원리와 섞여있었다. "정체성 제품 = 퀴즈"라는 Functional Fixedness (Duncker, 1945)에 갇혀있었고, McLuhan의 "매체가 메시지"를 무시했다. 원리 16개를 추출하고 퀴즈 구현 17개를 분리해서 YAML 1개 파일로 재구성했다. 이제 도메인도, 형태도 바꿀 수 있는 진짜 아이디어 공장이 됐다.
2. 퀴즈를 버리고 AI 대화형으로 전환했다
MBTI/퀴즈는 2025년에 포화. 케밋이 이미 하고 있음. 대신 AI 바리스타와 자연스러운 커피 대화 → 서프라이즈 성격 인사이트라는 새로운 형태를 채택했다.
3. "커피 × 데이팅"의 시장 기회를 확인했다
데이팅 앱 시장이 무너지고 있다 (Tinder -8%, Bumble -16% 구독자 감소). 스와이프 피로도 80%. 동시에 activity-based dating, IRL 만남이 급부상. Bumble이 파리에 카페를 열었지만, 커피에서 데이팅으로 가는 방향은 아무도 안 했다.
4. Progressive Disclosure + Trojan Horse Design을 배웠다
"커피 취향이 연애를 말해줍니다"를 첫 화면에 쓰면 — 트로이 목마 옆에 "안에 병사 있음" 현수막 단 거다. 첫 화면~Phase 1은 100% 커피. 연애는 Phase 2 서프라이즈에서만. "처음부터 알면 소음. 나중에 깨달으면 소름."
5. Lovable로 첫 프로토타입을 빌드했다
Base44(무료 크레딧 25개/월, Wix 종속)보다 Lovable(무료, UI 최강, GitHub 연동, 코드 소유)이 압도적으로 나았다. 프롬프트 하나로 5분 만에 랜딩 + 채팅 UI + 결과 카드까지 나왔다. 크레딧 2.8개 남음 — 신중하게 써야 함.
6. 빌드 도구의 발견
Claude Code/Codex = 로직 강하지만 디자인 약함. Lovable = 디자인 강하지만 로직은 프롬프트에 의존. 최적 조합: YAML로 로직 보관, Lovable로 UI 빌드, 필요시 코드 export 해서 Claude Code로 로직 강화.
7. 현실 인식
친구가 없어도 에브리타임 대행사 (₩9,900/대학)로 피드백 수집 가능. 하지만 그 전에 제품이 "애들 장난"이 아닌 수준이어야 한다. 오늘 Lovable 프로토타입이 그 기준선을 처음으로 넘겼다.
