## 오늘 한 것

### 1. SYNDI Phase 7 — Pretext Protocol 스트레스 테스트
- 4개 AI 모델(ChatGPT+Gemini vs Kimi+Claude Opus) 적대적 토론
- CRITIC 3.5 — CHALLENGER 1.5 승리
- 6개 쟁점 판결 → v2.1 스펙에 반영 (Axis Discovery Layer, Retention System, Cultural Portability 교정)
- 핵심 발견: 한국은 리스크 마켓이 아니라 세계 1위 타입 시스템 마켓 (40M+ 16P 응답)

### 2. COFFEE-BTI 제품 설계 (전부 잠금)
- 3축: 탐험↔루틴, 혼자↔함께, 분위기↔본질
- 8타입, 9문항, 6단계 궁합 엔진
- Domain Promise: "당신의 카페 취향이 당신과 맞는 사람을 말해준다"
- Product DNA JSON 완성 (SSOT)

### 3. 3-Way AI 빌드 실험
- 같은 Product DNA JSON → 코덱스, Claude Code, 안티그래비티 동시 빌드
- 결과: 안티그래비티 승 (바이럴 루프 + OG + E2E 검증)
- 코덱스 = 디자인 최강, Claude Code = 과도 설계(Zustand), 안티그래비티 = 제품 완성도 최강
- 학습 → Build Requirements Checklist로 고정

### 4. 독립 프로젝트 분리 + 배포
- 코덱스가 myceleb에서 coffee-bti 코드 추출 → `project-pretext/` 독립 빌드
- TypeScript 변환, 라우트 분리 (`/` → `/quiz` → `/result/[typeId]`)
- **Vercel 배포 완료: https://project-pretext.vercel.app**

### 5. 디자인 보강 (코덱스 2차)
- 뒤로가기 버튼, 결과 카드에 카페 추천 + 카페 습관 추가
- Playwright E2E 테스트 375px 통과
- 하지만 레이아웃 정렬/가독성 부족 → Claude Code 3차 브리프 작성

### 6. SYNDI v8.0.2 — 제품 컨셉 스트레스 테스트

## 오늘 배운 것

### Analysis Paralysis 패턴 인식
문서 12개, 배포 0개. 매번 "이제 만들자" 직전에 "큰 그림을 먼저..."로 빠지는 패턴 반복. 공식 설계, SYNDI 토론, 멀티모델 비교 — 전부 가치 있었지만, 실제 유저 데이터 0건인 상태에서 계속 이론만 쌓고 있었다.

### 업계 프레임워크 교차검증
- **YC (Michael Seibel):** "완전한 솔루션 유혹에 저항하라. Airbnb Lite는 한 달 안에 만들었다."
- **Google Design Sprint:** 5일 안에 프로토타입 → 유저 테스트
- **Basecamp Shape Up:** "배포 능력을 되찾은 후에 발견 프로세스를 개선해라"
- **Amazon Working Backwards:** PR/FAQ 먼저 — 하지만 대기업 프로세스, 솔로 빌더에겐 과도

4개 중 3개가 같은 말: **계획 최소화, 빠르게 만들어서, 실제 유저 앞에 내놓아라.**

### Soft Launch 전략
첫인상이 전부라서 "완벽해질 때까지" 기다리는 게 아니라, 첫인상을 보여줄 대상을 통제하는 것. Dropbox도 소수 그룹 베타로 시작.

### 수익화 타이밍
Phase 1에서 AdSense 붙이면 바이럴 엔진에 모래 뿌리는 것. 유저 5,000명 전까지 0원.
